Arbit 기능별 데이터베이스 구조
===============================

Arbit의 테이블을 다섯 가지 주요 기능에 따라 묶고, 기능별 데이터 흐름을 간단히 설명한다.
users와 events는 각각 사용자와 행사를 나타내는 중심 테이블이다.


1. 사용자 정보와 사용자가 직접 선택한 취향
--------------------------------------------

테이블

- users: 사용자 로그인 정보, 프로필, 거주 지역과 위치 정보
- user_categories: 사용자가 직접 선택한 관심 카테고리
- user_preference_events: 취향 설정 과정에서 사용자가 직접 선택한 선호 행사
- categories: 사용자 관심 카테고리와 행사 분류에 함께 사용하는 카테고리 목록

데이터 흐름

사용자가 가입하면 기본 정보를 users에 저장한다.
사용자가 관심 카테고리와 선호 행사를 선택하면 user_categories와 user_preference_events에 저장한다.
이 데이터는 사용자 초기 취향을 구성하고 추천 계산의 입력으로 사용한다.


2. 전시, 공연 등 행사 정보
--------------------------

테이블

- events: 행사 제목, 설명, 장소, 기간, 가격, 상태와 평균 평점
- categories: 전시, 공연, 축제 등 행사의 대표 분류
- age_restriction_keywords: 행사 연령 제한 값 목록
- event_age_restrictions: 행사와 연령 제한의 연결 정보
- classification_keywords: 행사 분위기와 세부 분류 키워드 목록
- event_classifications: 행사와 세부 분류 키워드의 연결 정보
- preference_keywords: 사용자 취향과 행사 특징을 표현하는 키워드 목록
- event_keywords: 행사와 취향 키워드의 연결 정보

데이터 흐름

행사 기본 정보와 대표 카테고리를 events와 categories에 저장한다.
연령 제한, 세부 분류와 취향 키워드는 각각의 키워드 및 연결 테이블에 저장한다.
저장된 행사 정보는 검색과 상세 화면에 제공되며, 사용자 활동의 대상과 추천 계산의 입력으로 사용한다.


3. 사용자의 북마크, 리뷰, 조회 및 클릭 기록
------------------------------------------

테이블

- bookmarks: 사용자가 북마크한 행사
- reviews: 사용자가 행사에 작성한 평점, 리뷰와 인증 이미지 정보
- event_detail_view_logs: 사용자의 행사 상세 조회 이력
- event_action_logs: 사용자의 상세 조회, 홈페이지 클릭과 행동 발생 위치 이력

데이터 흐름

사용자가 행사에 북마크, 리뷰, 조회 또는 클릭 행동을 하면 해당 활동 테이블에 기록한다.
리뷰 평점은 행사의 평균 평점에 반영한다.
누적된 활동 기록은 사용자의 관심도를 분석하고 사용자 키워드 가중치와 추천 결과를 갱신하는 데 사용한다.


4. 행사와 사용자의 특징을 표현하는 키워드
----------------------------------------

테이블

- keywords: 추천 계산에 사용하는 통합 키워드 목록
- user_keyword_weights: 사용자별 키워드 중요도
- event_keyword_weights: 행사별 키워드 관련도

데이터 흐름

사용자의 직접 선택과 활동 기록을 분석하여 user_keyword_weights를 생성하거나 갱신한다.
행사 기본 정보와 행사 분류 및 키워드를 분석하여 event_keyword_weights를 생성하거나 갱신한다.
사용자와 행사의 키워드 가중치를 비교한 결과를 추천 점수 계산에 사용한다.


5. 추천 계산 결과와 추천 실행 이력
----------------------------------

테이블

- recommendations: 사용자에게 현재 제공할 행사별 추천 점수와 추천 이유
- recommendation_runs: 추천 알고리즘 실행 단위와 알고리즘 및 모델 정보
- recommendation_items: 추천 실행별 행사 순위, 점수, 추천 이유와 상세 점수

데이터 흐름

사용자 취향, 활동 기록, 사용자 키워드 가중치와 행사 키워드 가중치를 입력으로 추천을 실행한다.
실행 정보는 recommendation_runs에 저장하고, 실행별 행사 결과는 recommendation_items에 저장한다.
현재 서비스에서 사용자에게 제공할 최신 추천 결과는 recommendations에 저장한다.
